# ICC Course Overview

## 5.1 Lesson 01

File: `Lesson01/lesson01.html`

Goal: Help students understand why problem discovery still matters in the AI era, then enter real scenarios and define a core pain point.

Core outputs:
- Scenario pain-point list
- One-sentence core pain point
- Four-question refinement notes

Project directions:
- Refrigerator food detection
- Orchard fruit disease inspection
- Office ineffective lighting monitoring
- Vehicle blind-spot visual reminder
- Rare bird smart observation system

## 5.1 Lesson 02

File: `Lesson02/lesson02.html`

Goal: Use design thinking and first principles to turn the core pain point into an invention topic draft, then use AI to support solution directions and evidence planning.

Core outputs:
- One-sentence invention topic draft
- 2-3 possible solution directions
- Evidence checklist
- Invention log framework

## 5.2 Lesson 03

Suggested file: `Lesson03/lesson03.html`

Goal: Explain how to learn technology in the AI era and establish basic understanding of hardware sensing, inputs, outputs, sensors, and actuators.

Suggested outputs:
- Possible sensing information needed by each project
- Initial sensor/input options
- Simple hardware exercise notes

## 5.2 Lesson 04

Suggested file: `Lesson04/lesson04.html`

Goal: Complete simple embedded-system and modeling exercises so students understand how a first prototype can be built.

Suggested outputs:
- Basic function breakdown
- Simple structure or appearance sketch
- Technical practice notes

## 5.3 Lesson 05

Suggested file: `Lesson05/lesson05.html`

Goal: Learn material selection and how to search, compare, and record parts on platforms such as Taobao.

Suggested outputs:
- Materials list
- First BOM draft
- Selection reasons

## 5.3 Lesson 06

Suggested file: `Lesson06/lesson06.html`

Goal: Explain how to systematically build a project by connecting problem, solution, materials, making steps, and presentation.

Suggested outputs:
- Project system diagram
- Making plan
- First presentation draft
