时代切片｜很多项目都从“看见异常”开始

许多真实项目不是一开始就做复杂机器，而是先把“容易忽略、来不及发现、看不清楚”的异常变成可观察的信号。

看见 → 判断 → 提醒
农业里，拍照识别病虫害已经出现

FAO 介绍过用图像识别帮助农户判断植物病虫害的应用。

建筑里，照明已经会跟着人和光变化

占用传感器、光照传感器可以让灯在不需要时自动关闭或调低亮度。

交通里，盲区提醒已经是安全技术

车辆能通过摄像头或接近传感器发现驾驶员不容易看到的区域。

生态保护里，自动相机能留下出现证据

摄像头和抓拍记录可以帮助巡护人员追踪鸟类出现、停留和活动。

看视频时别只看它用了什么技术，先听它把异常变成了什么用户价值。

设计思维｜HMW 问题

核心痛点之后，先问“我们如何……”

HMW 是 How Might We 的缩写，意思是“我们可以如何……”。它不是最终答案，而是让小组开始寻找方案的开放问题。

核心痛点：食材被忘记，发现时已经临期或变质。
→
我们如何帮助忙碌家庭及时发现临期或被遗忘的食材？
核心痛点：果农巡园时容易漏掉早期病斑。
→
我们如何帮助果农更早发现并记录果实病斑？
核心痛点：无人或自然光充足时仍然开灯。
→
我们如何帮助办公区发现并提醒无效照明？
核心痛点：停车场倒车时侧后方观察不足。
→
我们如何帮助司机更早发现车辆盲区风险？
核心痛点：珍稀鸟类出现和停留不容易被及时记录。
→
我们如何帮助巡护人员自动抓拍并记录珍稀鸟类活动？
核心痛点说明“哪里不够好”；HMW 打开“我们可以怎么改”。

方法拓展｜第一性原理

先问最本质要证明什么

第一性原理，就是暂时放下复杂外观，回到最基础、最关键的问题：这个项目至少要证明哪件事真的有用？

先证明一件最关键的事

冰箱：先证明“临期/遗忘提醒”能减少浪费。

果园：先证明“拍照记录 + 异常提示”能减少遗漏。

照明：先证明“无人/光线足够但灯亮”能被发现。

盲区：先证明“看不见的位置”能被视觉提醒。

鸟类：先证明“识别/抓拍 + 时间记录”能减少观察遗漏。

基础版只做一件最关键的事：能演示、能测试、能继续改。

五个小组｜基础版可以长什么样

先做一个能跑通的“小系统”

冰箱食材检测

输入：食材照片、标签或放入日期。

判断：是否临期、是否容易被忘记。

输出：灯光/屏幕/表格提醒。

果园病状检查

输入：果实照片、位置编号。

判断：颜色异常、斑点、破损记录。

输出：异常标记和巡检记录。

无效照明监控

输入：光照强度、是否有人。

判断：不需要开灯却仍开灯。

输出：区域提醒和节能记录。

盲区视觉提醒

输入：摄像头画面或模型盲区。

判断：风险物进入盲区。

输出：LED、蜂鸣或屏幕提示。

鸟类智慧观测

输入：摄像头画面、时间、停留区域。

判断：疑似目标鸟出现、停留或活动。

输出：抓拍照片和观测记录表。

小系统不等于小价值。它能证明一个关键判断，就能继续迭代。

基础版材料图

基础版通常不需要一开始就很复杂

标签 / 记录

食材名称、放入日期、临期提醒可以先用标签和表格跑通。

照片 / 位置

果园项目先记录“哪棵树、哪个位置、哪张照片”。

传感器 / 提醒

光照、人体感应、LED 或屏幕提示可以组成基础判断链。

模型 / 场景

盲区项目先用模型车、道路图和提醒灯说明风险位置。

相机 / 记录表

鸟类项目先用画面、时间和停留区域跑通抓拍记录。

能跑通一次完整演示，比堆很多功能更重要。

可直接使用的提示词
我的一句话选题初稿是： 【为了解决____在____遇到的____问题，设计一个能____的方案。】 请你不要直接替我决定方案。 请帮我列出： 1. 还需要追问用户的 5 个问题； 2. 可能的 3 个基础版方案； 3. 每个方案需要哪些材料或技术； 4. 课后应该收集哪些照片、访谈或已有产品截图。
1先问问题

让 AI 帮忙补盲点，而不是直接要“最佳方案”。

2再删想法

删掉太贵、太危险、太复杂、没法课堂测试的方案。

3最后留证据

把可验证的方案变成访谈、拍照、截图和测试任务。

AI 使用样例

把一个回答筛成课堂可做的任务

AI 可能给出

小组筛选后留下
  • 先做一个桌面模型：一间办公室、一个窗户、一盏灯。
  • 用“光照 + 是否有人”判断无效照明。
  • 只做提醒，不直接控制真实电路。
  • 记录 5 次测试：有人/无人、亮/暗、灯亮/灯灭。
留下来的方案要能回答：我能不能做？能不能测试？能不能证明有帮助？
